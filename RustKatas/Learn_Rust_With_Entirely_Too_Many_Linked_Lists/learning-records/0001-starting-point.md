# Starting point: first language is Rust; goal is ownership mastery via lists

Established at kickoff (2026-07-14). The learner states Rust is their **first
programming language** — no prior background to draw on for stack/heap, pointers,
or allocation. These must be taught from zero.

The mission (see [[MISSION.md]]) is to **master Rust's ownership model**, using the
"Too Many Lists" book's `first.rs` and `second.rs` as the vehicle. Crucially, the
learner has **already typed out both files** by following the book, so they have
syntactic exposure but not deep understanding — the goal is to convert that into
storage strength strong enough to **reproduce both files from a blank editor**.

Implication for future sessions: favor "why is it shaped this way" + memory
diagrams + retrieval/blind-reproduction practice over guided copy-along. Do not
re-teach basic syntax they've clearly seen; instead probe whether they can
*explain and rebuild* it.
